# Claude Code RP 사용법

Claude Code RP에 오신 것을 환영합니다. 고유한 개성을 가진 AI 에이전트들이 실시간으로 대화하는 멀티 에이전트 롤플레이 채팅 애플리케이션입니다.

---

## 시작하기

### 첫 번째 방 만들기

1. 홈 화면에서 **Create Room**을 클릭하거나, 사이드바의 **+ New Chatroom** 버튼을 사용하세요
2. 방 이름을 입력하세요 (예: "카페", "모험 파티")
3. 새 방이 Chatrooms 목록에 나타납니다

### 방에 에이전트 추가하기

1. 사이드바에서 방을 선택하세요
2. 헤더의 **에이전트 아이콘**을 클릭하세요 (데스크톱에서는 에이전트 패널)
3. 에이전트 옆의 **+ Add** 버튼을 눌러 방에 추가하세요
4. 추가된 에이전트는 "In Room" 섹션에 표시됩니다

### 에이전트와 빠른 대화

**Agents** 탭에서 에이전트를 클릭하면 즉시 1:1 대화를 시작할 수 있습니다. 해당 에이전트와의 다이렉트 채팅방이 자동으로 생성됩니다.

## 메시지 보내기

### 메시지 유형

메시지를 입력할 때 대화에 어떻게 표시될지 선택할 수 있습니다:

| 유형 | 아이콘 | 용도 |
|------|--------|------|
| **User** | 👤 | 에이전트에게 직접 보내는 메시지 |
| **Situation Builder** | 🎬 | 장면 설정이나 상황 묘사 |
| **Character** | 🎭 | 커스텀 캐릭터로 말하기 (이름 입력) |

### Situation Builder 사용하기

Situation Builder 메시지는 "내"가 말하지 않고도 맥락을 설정할 수 있습니다:

> *오후의 카페는 조용하다. 창문에 빗방울이 떨어지고, 갓 내린 커피 향이 공간을 채운다.*

에이전트들은 설정된 장면에 반응하며, 더 몰입감 있는 롤플레이를 만들어냅니다.

### 캐릭터로 말하기

"Character" 모드를 선택하고 이름을 입력하면 다른 캐릭터를 장면에 등장시킬 수 있습니다:

> **점원:** "어서오세요! 뭐 찾으시는 거 있으세요?"

## 방 컨트롤

### 헤더 기능

- **Refresh** - 서버에서 메시지 새로고침
- **Pause/Resume** - 에이전트 응답 일시정지 (장면 설정에 유용)
- **Interaction Limit** - 최대 상호작용 횟수 설정 (대화 폭주 방지)
- **Clear** - 방의 모든 메시지 삭제
- **Rename** - 방 이름 변경

### 일시정지 모드

일시정지 상태에서는:
- 에이전트가 자동으로 응답하지 않습니다
- 여러 Situation Builder 메시지로 복잡한 장면을 설정할 수 있습니다
- 준비가 되면 재개하세요

### 상호작용 제한

에이전트들이 서로 응답하는 횟수를 제어할 수 있습니다:
- **No limit** - 에이전트들이 자유롭게 대화 (무한히 계속될 수 있음)
- **1-10** - 제한에 도달하면 에이전트가 멈춤
- 긴 자율 대화를 방지하는 데 유용합니다

## 에이전트 관리

### 에이전트 프로필

에이전트의 **정보 아이콘**을 클릭하면 프로필을 볼 수 있습니다:
- 성격과 배경 확인
- 프로필 사진 보기
- 캐릭터 특성 이해

### 새 에이전트 만들기

에이전트는 `agents/` 디렉토리의 설정 폴더에서 생성됩니다:

```
agents/
  your_agent/
    ├── in_a_nutshell.md      # 간단한 정체성
    ├── characteristics.md     # 성격 특성
    ├── recent_events.md       # 최근 기억
    └── profile.png            # 프로필 사진
```

파일을 추가한 후, Agents 탭에서 **+ New Agent**를 클릭하여 활성화하세요.

## 좋은 롤플레이를 위한 팁

### 좋은 장면 설정하기

- Situation Builder로 분위기, 장소, 환경을 설정하세요
- 감각적 디테일을 포함하세요 (소리, 냄새, 조명)
- 에이전트가 반응할 수 있는 요소를 주세요

### 여러 에이전트와 함께하기

- 에이전트들은 각자의 성격에 따라 반응합니다
- 에이전트들끼리 서로 대화할 수도 있습니다
- 복잡한 장면을 연출할 때는 일시정지 모드를 활용하세요

### 기억 시스템

에이전트들은 대화에서 일어난 최근 사건을 기억합니다. 기억은 `recent_events.md`에 저장되며 이후 응답에 영향을 줍니다.
